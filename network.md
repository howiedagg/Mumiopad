### 一、 App 啟動 / 切回前景的連線判定流程

1. **偵測到啟動或切回前景**（狀態指示燈轉為：**🟡 橘燈 / 搜尋中**）
2. **檢查手機定位權限**：
   * **若未授權** ──> 跳過直連，直接進入下方 `[5. 階段二：網路廣播掃描]`。
   * **若已授權** ──> 讀取目前連線 Wi-Fi 的 BSSID (MAC位址)。
3. **比對本機資料庫**：
   * **若此 BSSID 沒有儲存紀錄** ──> 直接進入下方 `[5. 階段二：網路廣播掃描]`。
   * **若此 BSSID 有綁定電腦 UUID** ──> 進入下方 `[4. 階段一：捷徑直連]`。
4. **【階段一】捷徑直連（1.5 秒）**：嘗試直接連線該電腦上次已知的 IP：
   * **連線成功** ──> 進入 **🟢 綠燈 (已連線)**。
   * **連線失敗**（例如電腦關機或 IP 改變）──> 進入下一個步驟。
5. **【階段二】網路廣播掃描（5 秒）**：在目前 Wi-Fi 網域中發送廣播尋找：
   * **掃描成功並連上** ──> 進入 **🟢 綠燈 (已連線)**。
   * **掃描失敗** ──> 進入下一個步驟。
6. **【階段三】冷靜期重試（最長 3 分鐘）**：
   * 在 3 分鐘內，App 會間歇性地自動進行短暫搜尋。
   * **重試期間連上** ──> 進入 **🟢 綠燈 (已連線)**。
   * **3 分鐘結束仍失敗** ──> 停止搜尋，狀態進入 **⚪ 灰燈 (未連線)**。

---

### 二、 連線成功後的自動處理（🟢 綠燈）

* **步驟 1**：手機端啟用高效能、低延遲 Wi-Fi 鎖（防止操作過程中卡頓）。
* **步驟 2**：觸控板解除鎖定，允許手勢與滑鼠操作。
* **步驟 3**：自動儲存，將「目前 Wi-Fi 的 BSSID」與「此電腦的 UUID」在資料庫中綁定（作為下次的預設裝置）。

---

### 三、 切換視窗 / 進入背景的處理流程（⚪ 灰燈）

* **步驟 1**：偵測到 App 進入背景（如切換去回訊息、返回桌面）。
* **步驟 2**：主動關閉目前的 WebSocket 連線。
* **步驟 3**：釋放低延遲 Wi-Fi 鎖（節省手機電力）。

---

### 四、 手動點擊狀態膠囊與切換電腦流程

1. **點擊狀態膠囊**：彈出電腦管理清單（此時背景啟動 5 秒廣播更新列表，但不影響當前連線）。
2. **使用者手動點擊另一台已配對電腦**：
   * 狀態指示燈變更為：**🟡 橘燈**。
   * 主動斷開當前的連線 ──> 直接嘗試連線至新選擇的電腦。
   * **連線成功** ──> 進入 **🟢 綠燈**，並自動將「當前 Wi-Fi 的 BSSID」與「這台新電腦的 UUID」綁定儲存。
   * **連線失敗** ──> 狀態進入 **⚪ 灰燈**。

---

### 五、 初次連線（配對）流程

1. **無配對紀錄啟動 App**：顯示全螢幕引導畫面（狀態為 **⚪ 灰燈**）。
2. **手動開啟搜尋**：點擊「尋找電腦」展開搜尋清單。
3. **選擇目標**：點擊清單中顯示「未配對」的電腦（狀態指示燈轉為：**🟡 橘燈 / 配對中**）。
4. **發送配對請求**：手機發送配對訊號。
5. **電腦端授權**：電腦端彈出 Windows 對話框詢問是否允許：
   * **若點選「否」** ──> 配對失敗，狀態回到 **⚪ 灰燈**。
   * **若點選「是」** ──> 電腦生成安全憑證（Token）並回傳給手機。
6. **完成綁定**：手機儲存憑證與電腦 UUID ──> 連線成功進入 **🟢 綠燈**，並自動將「目前 Wi-Fi」與「此電腦」綁定儲存。
