# Mumiopad 連線協定與狀態判定規範

## 一、 App 啟動 / 切回前景的連線判定流程

當偵測到 App 啟動或從背景切回前景（生命週期進入 `ON_START`）時，系統會依據目前「本機信任名冊（Saved Servers）」是否為空，分流進入不同的工作模式：

### 1. 本機無信任紀錄（初次使用者 / 空名冊狀態）
* **決策**：不啟動自動連線協調器（Orchestrator），避免 NsdManager 資源佔用與衝突。
* **行為**：直接進入「五、 Mumiopad 初次連線（配對）與引導流程」，自動發起免點選的背景配對掃描。

### 2. 本機已有歷史信任紀錄（非初次連線）
* **決策**：啟動自動連線協調器（Orchestrator），狀態指示燈轉為 **🟡 橘燈 / 搜尋中**。
* **判定步驟**：
  1. **檢查手機定位權限**：
     * 若未授權 ──> 跳過直連，直接進入下方的 `[步驟 4：網路廣播掃描]`。
     * 若已授權 ──> 讀取目前連線 Wi-Fi 的 BSSID (MAC 位址)。
  2. **比對本機資料庫**：
     * 若此 BSSID 沒有儲存紀錄 ──> 直接進入下方的 `[步驟 4：網路廣播掃描]`。
     * 若此 BSSID 有綁定電腦 UUID ──> 進入 `[步驟 3：捷徑直連]`。
  3. **【階段一】捷徑直連（1.5 秒）**：嘗試直接連線該電腦上次已知的 IP：
     * 連線成功 ──> 進入 **🟢 綠燈 (已連線)**。
     * 連線失敗（例如電腦 IP 改變或關機）──> 進入下一個步驟。
  4. **【階段二】網路廣播掃描（5 秒）**：在目前 Wi-Fi 網域中發送 mDNS 廣播（尋找 `_mumiopad._tcp.`）搜尋信任名冊中的電腦：
     * 掃描成功並連上 ──> 進入 **🟢 綠燈 (已連線)**。
     * 掃描失敗 ──> 進入下一個步驟。
  5. **【階段三】冷靜期重試（最長 3 分鐘）**：
     * 在 3 分鐘之內，App 會間歇性自動進行短暫搜尋（每隔 10 秒發起一次 2 秒的短暫監聽）。
     * 重試期間連上 ──> 進入 **🟢 綠燈 (已連線)**。
     * 3 分鐘結束仍失敗 ──> 停止搜尋，狀態進入 **⚪ 灰燈 (未連線)**。

## 二、 連線成功後的自動處理（🟢 綠燈）

當與電腦端伺服器成功建立 WebSocket 連線（接收到 `auth_ok` 或完成 `pair_success`）時，系統自動執行以下優化與資料綁定：

* **優化 1**：手機端啟用高效能、低延遲 Wi-Fi 鎖（持鎖防止操作過程中封包卡頓）。
* **優化 2**：觸控板解除鎖定，允許手勢、滑鼠與鍵盤操作。
* **優化 3**：更新電腦上次已知 IP（`lastKnownIp`），為下次的「捷徑直連」提供依據。
* **優化 4**：自動儲存，將「目前連線 Wi-Fi 的 BSSID」與「此電腦的 UUID」在資料庫中綁定（作為下次此地點的預設裝置）。

## 三、 切換視窗 / 進入背景的處理流程（⚪ 灰燈）

當偵測到 App 進入背景（生命週期進入 `ON_STOP`）時，為節省手機電力並釋放系統資源，執行以下重設：

* **步驟 1**：主動關閉當前的 WebSocket 連線。
* **步驟 2**：釋放低延遲 Wi-Fi 鎖（回復系統省電狀態）。
* **步驟 3**：執行 `orchestrator.stop()` 停止一切背景自動搜尋，並利用「協程取消機制（invokeOnCancellation）」確保底層 `NsdManager` 的掃描監聽器被立馬登出，避免殘留。

## 四、 手動點擊狀態膠囊與切換/管理電腦流程

此流程僅在「本機已有信任紀錄」時啟用。使用者可隨時點擊狀態列膠囊，彈出對話框管理信任名冊：

### 1. 開啟管理對話框
* 背景自動暫停 `orchestrator.stop()`（釋放 NsdManager），並同時啟動 5 秒的手動配對掃描 `startPairingScan()` 以刷新列表。
* 列表分為「已信任的電腦（帶有綠色狀態點）」與「新發現未配對的電腦（帶有藍色狀態點）」。

### 2. 使用者手動點擊另一台已配對電腦
* 狀態指示燈變更為：**🟡 橘燈**。
* 主動斷開當前的連線 ──> 直接嘗試連線至新選擇的電腦（手動直連）。
* 連線成功 ──> 進入 **🟢 綠燈**，自動綁定當前 BSSID，關閉選單並重啟協調器。
* 連線失敗 ──> 狀態進入 **⚪ 灰燈**。

### 3. 使用者刪除特定已配對電腦
* 若被刪除的電腦是當前正連線的電腦，主動發送 `unpair` 註銷請求，中斷連線並設定為未連線狀態。
* 將該電腦自 PairingManager 與地點綁定紀錄中抹除。
* **刪除後的動態判定**：
  * **若信任清冊已空**：自動關閉管理選單對話框，手機畫面自動切換回「五、 Mumiopad 初次連線引導流程」，並啟動配對掃描。
  * **若信任清冊仍有其他電腦**：保持管理選單開啟，重啟連線協調器 `orchestrator.start()` 去自動尋找其他剩餘電腦，並同時調用 `startPairingScan()` 刷新清單。

## 五、 Mumiopad 初次連線（配對）與引導流程

當本機無任何配對紀錄時，App 自動進入沉浸式引導。此流程**不使用任何對話框彈窗**，而是直接在主螢幕上透過流暢的內嵌式（Inline）轉場引導使用者：

### 1. 【自動探索階段】
* **觸發**：啟動 App 後自動發起。
* **介面**：顯示溫和的圓形載入動畫，文字提示為：
  * 主標題：`正在尋找電腦...`
  * 副標題：`請確保已在電腦上啟動伺服器，且處於相同 Wi-Fi 網路`
* **邏輯**：背景持續發起 `startPairingScan()`，無 NsdManager 衝突。

### 2. 【卡片選擇階段】
* **觸發**：當 mDNS 搜尋到第一台未配對電腦時自動轉場。
* **介面**：載入動畫淡出，顯現出代表電腦主機的精緻圓角卡片（顯示實際電腦名稱，例如 `Howie-PC`），下方帶有小字 `點擊配對`。
* **邏輯**：卡片本身即為唯一互動點，無多餘的「配對」按鈕。

### 3. 【等待授權階段】
* **觸發**：使用者點擊電腦卡片。
* **介面**：其他卡片淡出，該卡片放大居中，顯示引導文字：
  * 主標題：`已向電腦發送連線請求`
  * 副標題：`請在電腦螢幕上點選「是」以允許連線`
  * 底部提供一個優雅的「取消」按鈕，點擊可取消連線並安全回歸【卡片選擇階段】。
* **電腦端伺服器行為**：
  * 接收到手機的 `pair_request`，在電腦螢幕中央彈出 Windows 對話框，標題為 `Mumiopad 配對請求`，內容詢問是否允許手機控制。
  * **若使用者點選「否」**：伺服器回傳 `pair_fail` 拒絕連線，手機端引導畫面顯示連線被拒絕。
  * **若使用者點選「是」**：伺服器隨機生成一組安全 Token，儲存於 `server_config.json`，並向手機回傳包含電腦主機名稱與 Token 的 `pair_success` 封包。

### 4. 【綁定與啟用階段】
* **觸發**：手機收到 `pair_success`。
* **行為**：手機儲存該 Token 與電腦 UUID，將目前 Wi-Fi 綁定此電腦，引導畫面優雅退場（淡出），無縫顯現出觸控板主介面，狀態燈轉為 **🟢 綠燈** 並觸發短暫震動，完成初次連線。
